# Commit Message Format

## When to apply

При формировании commit-сообщения по запросу пользователя — любая формулировка с этим намерением: «напиши комментарий к коммиту», «сформируй commit message», «оформи коммит», «commit message please» и т.п. Распознаётся по intent, не по literal-фразе.

## Format contract

Сообщение коммита состоит из заголовка и трёх обязательных секций.

### Title line

```
<type>(<scope>): <короткое summary на русском>
```

- `<type>`: один из `feat | fix | refactor | perf | chore | docs | test`.
- `<scope>`: краткое имя логического модуля (Database, WS, Funding, Arbitrage, AdapterGate, Core и т.п.).
- Title — повелительное наклонение, ≤ 120 символов, без точки в конце.

### Body sections (обязательные)

Все три секции присутствуют всегда. Если для секции нет содержания — поставить «—» вместо пустых буллетов.

```
Решение:
- <bullet list: что было изменено концептуально>
- <почему это решает проблему>

Изменения:
- <file_path>:
  - <конкретные методы / логика>
- <следующие файлы…>

Эффект:
- <итоговое поведение и гарантии>
- <практическое влияние на корректность / стабильность / производительность>
```

### Style notes (рекомендации, не инварианты)

- Язык: русский для summary и буллетов; `<type>`, `<scope>`, имена файлов/идентификаторов — в исходной форме (английский, conventional-commit).
- Маркеры списка — только `-`.
- Не упоминать AI / диалог / «этот чат».
- Готовый текст подходит для прямой вставки в `git commit -m`.

## Anti-examples

- `Реализовал новый функционал` — нет type/scope, нет bullet-секций.
- `feat: Refactored…` (с точкой; смешанный язык в title).
- Пустая «Изменения:» с плейсхолдерами `<file_path>` — не оставлять, либо убрать секцию (лучше — поставить «—»).

## Worked example

```
fix(WS): убрать утечку соединения OKX при reconnect

Решение:
- Добавлен явный `close()` для старого WebSocket перед открытием нового.
- Зафиксирован race condition между cleanup и reconnect-таймером.

Изменения:
- electron/engine/arbitrage/managers/WebSocketManager.ts:
  - reconnect(): добавлен await this.cleanup() перед new WebSocket().
- electron/engine/arbitrage/adapters/OKXAdapter.ts:
  - onClose handler: snapshot connId до cleanup, чтобы не путать сессии.

Эффект:
- Утечка сокетов устранена (проверено по lsof в стабильном тесте 1 час).
- Reconnect стабильно отрабатывает; race condition устранён.
- Стабильность WS-соединения ↑, нагрузка на CPU ↓ (меньше zombie-сокетов).
```

## Notes

- Это format contract, не safety инвариант. Деviate с обоснованием допустимо (например, для merge-коммитов или revert — другой шаблон).
- При отсутствии данных для секции — пишем «—» (не оставляем плейсхолдеры в треугольных скобках).
